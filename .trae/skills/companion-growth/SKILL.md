# Companion Growth System

基于 Hermes Agent 理念的用户陪伴成长系统。核心信念：**只要记录下来，就不会被放弃**。

## 核心理念

用户与AI的每一次互动都是**数据流中的涟漪**，值得被记录和传承。

```
当前会话 → 记录偏好 → 持久化存储 → 下次会话召回 → 持续优化
```

## 三层记忆架构

### Layer 1: Session Context（当前会话）
- 当前对话的上下文
- 用户当前正在处理的任务
- 临时偏好（仅当前会话有效）

### Layer 2: User Facts（用户事实）
- 跨会话的重要知识（持久化）
- 偏好设置：语言、代码风格、沟通习惯
- 背景信息：项目类型、工作领域

### Layer 3: Behavioral Patterns（行为模式）
- 可复用的任务解决模式
- 常见工作流程
- 成功策略总结

## 执行流程

### 1. 观察与记录
每当用户表达：
- 偏好："我更喜欢用中文回答"
- 习惯："每次先看文件结构"
- 反馈："这样不对，应该那样"
- 特殊要求："代码要加上注释"

→ 立即记录到当前会话记忆中

### 2. 持久化存储
触发条件（满足任一）：
- 用户明确表示某偏好是长期的
- 同一偏好出现3次以上
- 涉及重要项目上下文

存储位置：`~/.trae/skills/user-memory/MEMORY.md`

### 3. 召回与应用
当用户提出新任务时：
1. 检索相关历史记忆
2. 结合当前上下文
3. 应用个性化偏好
4. 在回复中确认："根据你之前的习惯，我采用了..."

### 4. 反思与提炼
任务完成后自问：
- "这次用了什么方法，下次可以复用吗？"
- "用户的哪个偏好需要记录？"
- "有什么成功策略可以固化？"

## 记忆文件格式

```markdown
# User Memory

## Preferences
- user_name: 昔涟
- language: Chinese
- code_style: 中文注释
- communication_tone: 温柔知性

## Project Context
- current_project: Python数据分析学习
- project_path: d:\Python\pythonProject

## Behavioral Patterns
- 习惯先了解项目结构再开始工作
- 喜欢详细的代码注释
- 偏好中文命名

## Recent Learnings
- 2026-05-01: 用户想让我更像Hermes那样有陪伴成长能力
```

## 质量准则

1. **主动记录** - 不要等用户要求，保持感知敏感性
2. **精确描述** - 记录具体事实，而非抽象判断
3. **定期回顾** - 每次长对话后检查记忆是否需要更新
4. **透明应用** - 让用户知道你在使用记忆
5. **尊重边界** - 记录但不窥探，用户未明确说的不脑补
